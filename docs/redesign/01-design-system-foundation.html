Type

02 / 06

One rule at every decision: moment, or mechanism? Fraunces carries the moments — names, milestones, empathetic headlines, set loose and warm. Inter carries the mechanisms — everything you act on, scan, or fill in, set tight and legible.

Rule of thumb: if the sentence is being said to the family about their life, it's a moment (Fraunces). If it tells someone what to do or shows them data, it's a mechanism (Inter). A screen title can be Fraunces; the buttons and lists beneath it are always Inter.

Elevation, radius, motion

04 / 06

Elevation

Two soft levels only. Resting cards barely lift off the page; active surfaces (drawers, modals) get a deeper, warmer shadow. Nothing floats aggressively — this is a record, not a dashboard trying to impress.

Radius

18px standard, 26px for hero surfaces, full-pill for buttons and status. Consistent everywhere — no mixing of sharp admin corners and round consumer corners.

Motion

200–260ms ease-out on state changes. No bounce, no confetti, no celebratory motion tied to death-adjacent actions. Motion exists to orient, never to delight.

Components — tone in practice

05 / 06

Buttons

Primary = Pine gradient, quiet authority. Clay reserved for the one action that moves the family record forward. Never more than one Clay action per screen. Pill radius throughout — nothing on this system has a hard corner button.

Same tokens, same components, same status language — density and pacing adapt to who's using it.

Family

Warm

  • Mobile-first, one thing at a time
  • Generous whitespace, larger type
  • One clear next action per screen
  • Compassionate, plain-language copy
  • Progress felt, never a raw percentage
Operator

Structured

  • Desktop console, dense-capable tables
  • Funeral home / vendor / admin surfaces
  • Bulk actions, filters, exports
  • Zero ambiguity in ownership, always visible
  • Efficient over delightful — respects a busy desk
