# Passage Beta and Demo Operator Plan

## Objective

Use Codex as a low-intervention product/growth operator that keeps Passage moving toward pilots, demos, and customer learning while the owner makes only high-leverage decisions.

## Weekly Rhythm

Monday:
- Review production QA and open risks.
- Pick 3 build tasks and 2 growth tasks.

Wednesday:
- Run persona QA.
- Update demo script and objections.

Friday:
- Summarize what shipped.
- Prepare next outreach assets.
- List owner decisions.

## Demo Assets To Maintain

- Funeral home 15-minute demo script.
- Local one-location demo case.
- Multi-location demo case.
- Family view from funeral home.
- Export CSV walkthrough.
- Act-on-behalf walkthrough.
- Vendor request walkthrough.
- Pricing and pilot offer explanation.

## Beta Pipeline

Track leads in a simple table or CRM with:
- Name
- Type: family, funeral home, vendor, clergy/chaplain, attorney
- Source
- Status
- Last touch
- Objection
- Next step

## Funeral Home Sales Narrative

Core promise:
Passage reduces repeated calls, gives families a calmer command center, and lets staff act on behalf of families without losing proof.

Demo arc:
1. Create or load a case.
2. Show today's value metrics.
3. Open family case.
4. Act on behalf of family.
5. Show family status view.
6. Export data for their current system.
7. Explain pilot billing.

## Vendor Network Narrative

Core promise:
Trusted local help appears only when it helps complete a family task.

Rules:
- No directory.
- No ads.
- No endless lists.
- Funeral homes can control preferred partners.
- Passage tracks requests and future revenue opportunities.

## Owner Approval Gates

Codex can draft but should not execute:
- Customer emails
- Vendor outreach
- Funeral home outreach
- Production SQL
- Pricing changes
- Legal/compliance copy
